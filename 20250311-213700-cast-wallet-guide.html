命令详解

1. 创建新钱包

# 生成新私钥（直接输出到控制台）
cast wallet new

# 保存到 Keystore（加密存储，需要密码）
cast wallet new myaccount

# 生成助记词（12 词）
cast wallet new-mnemonic

# 生成 24 词助记词
cast wallet new-mnemonic --words 24

参数：

  • --unsafe-password <PASSWORD> - 非交互式设置密码（仅用于脚本）

2. 靓号生成

# 生成以 0xabc 开头的地址
cast wallet vanity --starts-with 0xabc

# 生成以 dead 结尾的地址
cast wallet vanity --ends-with dead

# 同时满足前后缀（更慢）
cast wallet vanity --starts-with 0xabc --ends-with dead

参数：

  • --starts-with <HEX> - 地址前缀
  • --ends-with <HEX> - 地址后缀
  • --nonce <NONCE> - 指定起始 nonce（用于合约地址预测）

3. 导入私钥

# 交互式导入（输入私钥，设置密码）
cast wallet import myaccount --private-key 0x...

# 从文件导入
export PK=$(cat privatekey.txt)
cast wallet import myaccount --private-key "$PK"

# 非交互式（脚本用，注意安全性）
cast wallet import myaccount --private-key 0x... --unsafe-password "mypassword"

参数：

  • --private-key <KEY> - 要导入的私钥
  • --unsafe-password <PASSWORD> - 跳过密码提示
  • --keystore-dir <PATH> - 指定 Keystore 目录（默认 ~/.foundry/keystores）

4. 列出账户

# 列出所有 Keystore 账户
cast wallet list

# 指定目录
cast wallet list --keystore-dir /path/to/keystores

5. 地址转换

# 从私钥获取地址
cast wallet address --private-key 0x...

# 从 Keystore 获取地址
cast wallet address --account myaccount

# 从助记词派生
cast wallet derive "word1 word2 ... word12"

# 派生特定路径（默认 m/44'/60'/0'/0/0）
cast wallet derive "word1 word2 ..." --path "m/44'/60'/0'/0/1"

参数：

  • --private-key <KEY> - 指定私钥
  • --account <NAME> - 使用 Keystore 中的账户
  • --keystore-dir <PATH> - 指定 Keystore 目录

6. 签名消息

# 签名普通消息
cast wallet sign --private-key 0x... "Hello World"

# 签名 hash（不进行 personal_sign 包装）
cast wallet sign --no-hash --private-key 0x... 0xabcd...

# 使用 Keystore 账户签名
cast wallet sign --account myaccount "Hello"

# 签名 EIP-712 类型数据
cast wallet sign --data $(cast abi-encode "...") --private-key 0x...

# EIP-7702 授权签名
cast wallet sign-auth --private-key 0x... --contract 0xDelegator --nonce 0

参数：

  • --private-key <KEY> - 使用指定私钥
  • --account <NAME> - 使用 Keystore 账户
  • --no-hash - 直接对输入进行签名（不添加前缀）
  • --data - 签名类型化数据

7. 验证签名

# 验证消息签名
cast wallet verify --address 0x... "Hello World" 0x<signature>

# 验证 hash 签名
cast wallet verify --no-hash --address 0x... 0x<hash> 0x<signature>

8. 解密 Keystore

# 导出私钥（交互式）
cast wallet decrypt-keystore myaccount

# 从指定目录
cast wallet decrypt-keystore myaccount --keystore-dir /custom/path

# 非交互式（脚本用）
cast wallet decrypt-keystore myaccount --unsafe-password "mypassword"

    "cipher": "aes-128-ctr",

配置示例

场景 1: 快速测试（本地 Anvil）

# 生成测试账户
cast wallet new test-account

# 导出私钥给 Anvil
PK=$(cast wallet decrypt-keystore test-account --unsafe-password "testpass")

# 启动 Anvil 时预充值
anvil --fork-url $MAINNET_RPC --accounts 10 \
  --balance 10000 \
  --private-key $PK

场景 2: 生产环境部署脚本

#!/bin/bash
# deploy.sh

# 从环境变量读取
export ETH_KEYSTORE="${HOME}/.foundry/keystores"
export ETH_KEYSTORE_ACCOUNT="deployer"
export ETH_PASSWORD="${DEPLOYER_PASSWORD}"  # CI/CD 注入

# 验证地址
echo "Deploying from: $(cast wallet address --account deployer)"

# 使用 forge 部署
forge script scripts/Deploy.s.sol --rpc-url $RPC_URL --broadcast --account deployer

场景 3: 与 JS/TS 集成

// 读取 cast keystore 并在 JS 中签名
import { Wallet } from 'ethers';
import fs from 'fs';

// 方法 1: 直接用 ethers 解密
const keystorePath = `${process.env.HOME}/.foundry/keystores/myaccount`;
const keystore = fs.readFileSync(keystorePath, 'utf8');
const wallet = await Wallet.fromEncryptedJson(keystore, password);

// 方法 2: 用 cast 导出私钥再导入

  `cast wallet decrypt-keystore myaccount --unsafe-password "${password}"`,
  { encoding: 'utf8' }

const wallet2 = new Wallet(privateKey);

常见问题

Q: 如何修改 Keystore 密码？

# cast 没有直接的修改密码命令，需要：
# 1. 解密导出私钥
PK=$(cast wallet decrypt-keystore myaccount --unsafe-password oldpass)
# 2. 删除旧文件
rm ~/.foundry/keystores/myaccount
# 3. 重新导入
export PK
cast wallet import myaccount --private-key "$PK" --unsafe-password newpass

Q: Keystore 文件丢了怎么办？

  • 如果有助记词：用 cast wallet derive 重新派生
  • 如果有私钥备份：用 cast wallet import 重新导入
  • 什么都没有：无法恢复

Q: 多个项目如何管理不同账户？

# 项目 A 的配置
export FOUNDRY_PROFILE=project-a
export ETH_KEYSTORE="${HOME}/.foundry/keystores/project-a"
mkdir -p $ETH_KEYSTORE
cast wallet new project-a-deployer --keystore-dir $ETH_KEYSTORE

# 在 foundry.toml 中使用
# [profile.project-a]
# eth_keystore = "~/.foundry/keystores/project-a"

Q: 命令行中如何安全传递密码？

不推荐：--unsafe-password 会暴露在进程列表中

推荐：

  • 使用环境变量 ETH_PASSWORD
  • 或使用 --password-file（cast 某些命令支持）
  • 或使用交互式输入（人工操作时）

Q: 如何备份 Keystore？

# 备份整个目录
tar czvf foundry-keystores-backup.tar.gz ~/.foundry/keystores/

# 同时务必备份密码！没有密码无法解密
