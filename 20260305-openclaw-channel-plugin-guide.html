1. 概述与架构

1.1 什么是 Channel Plugin？

Channel Plugin 是 OpenClaw 的扩展模块，用于将 OpenClaw 连接到各种消息平台（如微信、钉钉、飞书、Slack 等）。通过 Channel Plugin，你可以让 AI Agent 在这些平台上接收消息并发送回复。

1.2 架构概览

┌─────────────────────────────────────────────────────────────────┐
│                        外部消息平台                              │
│           (微信/钉钉/飞书/Slack/Discord/微博...)                  │
└───────────────────────────┬─────────────────────────────────────┘
                            │ HTTP/WebSocket/Webhook
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│                     Channel Plugin 层                            │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐          │
│  │   输入适配器  │  │   账户管理   │  │   输出适配器  │          │
│  │  (Inbound)   │  │  (Account)   │  │  (Outbound)  │          │
│  └──────────────┘  └──────────────┘  └──────────────┘          │
└───────────────────────────┬─────────────────────────────────────┘
                            │ api.receiveMessage()
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│                     OpenClaw Gateway                             │
│              (消息路由 + AI Agent 调度)                          │
└───────────────────────────┬─────────────────────────────────────┘
                            │ outbound.sendText()
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│                     Channel Plugin 层                            │
│                    (将回复发送到平台)                            │
└─────────────────────────────────────────────────────────────────┘

1.3 核心组件

组件 职责 关键 API
Inbound (输入) 接收外部平台消息，转发给 Gateway api.receiveMessage()
Outbound (输出) 接收 Gateway 回复，发送到外部平台 outbound.sendText()
Account (账户) 管理多账户配置和状态 config.listAccountIds()
Config (配置) 读取用户配置的参数 api.config.get()

步骤 1：创建插件元数据

创建 openclaw.plugin.json：

{
  "id": "my-channel",
  "name": "My Channel Plugin",
  "version": "1.0.0",
  "entry": "./src/index.ts",
  "configSchema": {
    "type": "object",
    "properties": {
      "apiKey": { "type": "string" },
      "webhookPort": { "type": "number", "default": 3000 }
    }
  }
}
步骤 2：编写主入口文件

创建 src/index.ts：

import type { PluginAPI } from "openclaw/plugin-sdk/core";

export default function register(api: PluginAPI) {
  const logger = api.logger;
  const config = api.config.get();
  
  logger.info("My Channel Plugin 启动中...");

  // 注册 Channel
  api.registerChannel({
    plugin: {
      id: "my-channel",
      
      meta: {
        id: "my-channel",
        label: "My Channel",
        selectionLabel: "My Channel (Custom)",
        blurb: "自定义 Channel 插件示例",
      },
      
      capabilities: {
        chatTypes: ["direct", "group"],
      },
      
      config: {
        listAccountIds: (cfg) => 
          Object.keys(cfg.channels?.["my-channel"]?.accounts ?? {}),
        
        resolveAccount: (cfg, accountId) => 
          cfg.channels?.["my-channel"]?.accounts?.[accountId ?? "default"],
      },
      
      outbound: {
        deliveryMode: "direct",
        
        sendText: async ({ text, chatId, accountId }) => {
          logger.info(`发送消息到 ${chatId}: ${text}`);
          // 在这里实现消息发送逻辑
          return { ok: true };
        },
      },
    },
  });
}

# 重启 Gateway

3. 核心概念

3.1 Plugin API 详解

api 对象

interface PluginAPI { logger: Logger; // 日志记录器 config: ConfigAPI; // 配置访问 registerChannel(opts: ChannelRegistration): void; registerGatewayMethod(name: string, handler: Function): void; registerCommand(cmd: Command): void; registerService(service: Service): void; }

api.receiveMessage() - 接收消息

当外部平台有消息时，调用此方法将消息传递给 OpenClaw Gateway：

await api.receiveMessage({
  channel: "my-channel",        // Channel ID（必需）
  accountId: "default",         // 账户 ID（多账户场景）
  chatType: "direct",           // 聊天类型: direct | group | channel
  chatId: "user_123",           // 对话/群组 ID（必需）
  senderId: "user_123",         // 发送者 ID
  senderName: "张三",            // 发送者昵称（可选）
  body: "你好",                  // 消息内容（必需）
  messageId: "msg_001",         // 消息 ID（用于回复追踪）
  timestamp: Date.now(),        // 时间戳（毫秒）
  
  // 媒体消息（可选）
  media: {
    type: "image",              // image | audio | video | file
    url: "https://example.com/img.jpg",
    mimeType: "image/jpeg",
    fileName: "photo.jpg",
    fileSize: 1024000,
  },
  
  // 引用回复（可选）
  replyTo: {
    messageId: "msg_000",
    body: "被引用的消息内容",
  },
  
  // 群组信息（可选）
  groupInfo: {
    groupId: "group_123",
    groupName: "测试群",
  },
});

outbound.sendText() - 发送消息

Gateway 生成回复后，会调用此方法：

outbound: {
  deliveryMode: "direct",
  
  sendText: async (params) => {
    const {
      text,           // 回复文本内容
      chatId,         // 目标对话 ID
      accountId,      // 账户 ID
      senderId,       // 发送者 ID（原消息发送者）
      messageId,      // 原消息 ID（用于回复）
      threadId,       // 线程/话题 ID
      channelData,    // 通道特定数据
    } = params;
    
    // 实现发送逻辑
    await sendToExternalPlatform(chatId, text);
    
    return { 
      ok: true, 
      messageId: "reply_001"  // 可选：返回发送的消息 ID
    };
  },
}

5. 多账户管理

5.1 为什么需要多账户？

  • 环境隔离：生产环境 vs 测试环境
  • 多租户：为不同客户/团队提供独立服务
  • 故障转移：主账户异常时切换到备用账户
  • 负载均衡：分散请求到多个账户

5.2 配置结构

{
  "channels": {
    "my-channel": {
      "accounts": {
        // 账户 1：生产环境
        "production": {
          "enabled": true,
          "apiKey": "prod_key_xxx",
          "apiSecret": "prod_secret_xxx",
          "webhookPort": 3000
        },
        // 账户 2：测试环境
        "staging": {
          "enabled": true,
          "apiKey": "staging_key_xxx",
          "apiSecret": "staging_secret_xxx",
          "webhookPort": 3001
        },
        // 账户 3：备用
        "backup": {
          "enabled": false,  // 默认禁用
          "apiKey": "backup_key_xxx",
          "apiSecret": "backup_secret_xxx"
        }
      }
    }
  }
}

5.3 多账户实现代码

import type { PluginAPI } from "openclaw/plugin-sdk/core";
import { randomUUID } from "crypto";

interface AccountConfig {
  enabled: boolean;
  apiKey: string;
  apiSecret: string;
  webhookPort: number;
}

export default function register(api: PluginAPI) {
  const logger = api.logger;
  const config = api.config.get();
  
  // 存储每个账户的连接
  const accountConnections = new Map();
  
  // ========== 获取所有启用的账户 ==========
  const channelConfig = config.channels?.["my-channel"];
  const accounts: Record = channelConfig?.accounts ?? {};
  
  const enabledAccounts = Object.entries(accounts)
    .filter(([_, cfg]) => cfg.enabled)
    .map(([id]) => id);
  
  logger.info(`找到 ${enabledAccounts.length} 个启用的账户: ${enabledAccounts.join(", ")}`);
  
  // ========== 为每个账户创建连接 ==========
  for (const accountId of enabledAccounts) {
    const accountCfg = accounts[accountId];
    
    // 创建连接（例如 WebSocket、HTTP 客户端等）
    const connection = createConnection(accountId, accountCfg);
    accountConnections.set(accountId, connection);
    
    // 设置消息接收处理器
    setupMessageHandler(accountId, connection, api);
  }
  
  // ========== 创建连接 ==========
  function createConnection(accountId: string, cfg: AccountConfig) {
    logger.info(`[${accountId}] 创建连接...`);
    
    // 示例：创建 WebSocket 连接
    // const ws = new WebSocket(`wss://api.example.com?key=${cfg.apiKey}`);
    
    // 示例：创建 HTTP 服务器接收 Webhook
    // const app = express();
    // app.listen(cfg.webhookPort);
    
    return {
      accountId,
      config: cfg,
      // ws 或 http 实例
    };
  }
  
  // ========== 消息接收处理 ==========
  function setupMessageHandler(accountId: string, connection: any, api: PluginAPI) {
    // 当收到外部消息时
    const handleMessage = async (externalMsg: any) => {
      try {
        // 转换为 OpenClaw 消息格式
        const message = {
          channel: "my-channel",
          accountId: accountId,           // 关键：标识来自哪个账户
          chatType: externalMsg.isGroup ? "group" : "direct",
          chatId: externalMsg.chatId,
          senderId: externalMsg.userId,
          senderName: externalMsg.nickname,
          body: externalMsg.text,
          messageId: externalMsg.id || randomUUID(),
          timestamp: Date.now(),
        };
        
        // 发送到 Gateway
        await api.receiveMessage(message);
        
      } catch (error) {
        logger.error(`[${accountId}] 处理消息失败:`, error);
      }
    };
    
    // 绑定到连接（根据实际连接类型）
    // connection.ws.on("message", handleMessage);
    // 或
    // connection.app.post("/webhook", (req, res) => {
    //   handleMessage(req.body);
    //   res.json({ ok: true });
    // });
  }
  
  // ========== 注册 Channel ==========
  api.registerChannel({
    plugin: {
      id: "my-channel",
      
      meta: {
        id: "my-channel",
        label: "My Channel",
        selectionLabel: "My Channel (多账户支持)",
        blurb: "支持多账户的 Channel 插件",
      },
      
      capabilities: {
        chatTypes: ["direct", "group"],
      },
      
      config: {
        // 列出所有账户
        listAccountIds: (cfg) => {
          return Object.keys(cfg.channels?.["my-channel"]?.accounts ?? {});
        },
        
        // 解析账户配置
        resolveAccount: (cfg, accountId) => {
          const accounts = cfg.channels?.["my-channel"]?.accounts ?? {};
          return accounts[accountId ?? "default"];
        },
      },
      
      outbound: {
        deliveryMode: "direct",
        
        sendText: async (params) => {
          const { text, chatId, accountId } = params;
          
          // 找到对应账户的连接
          const connection = accountConnections.get(accountId);
          if (!connection) {
            logger.error(`[${accountId}] 未找到连接`);
            return { ok: false, error: "Account not connected" };
          }
          
          // 实现发送逻辑
          try {
            // await connection.api.sendMessage(chatId, text);
            logger.info(`[${accountId}] 发送消息到 ${chatId}: ${text}`);
            return { ok: true };
          } catch (error) {
            logger.error(`[${accountId}] 发送失败:`, error);
            return { ok: false, error: String(error) };
          }
        },
      },
    },
  });
}

6. 消息类型处理

6.1 支持的消息类型

类型 描述 Content-Type
text 纯文本消息 text/plain
image 图片（JPG/PNG/GIF） image/*
audio 语音/音频 audio/*
video 视频 video/*
file 任意文件 application/*
location 地理位置 -
sticker 表情包/贴图 image/*

6.2 接收媒体消息

async function handleIncomingMessage(externalMsg: any) {
  let media = undefined;
  
  // 处理图片消息
  if (externalMsg.type === "image") {
    media = {
      type: "image",
      url: externalMsg.imageUrl,           // 图片 URL
      mimeType: "image/jpeg",
      fileName: "image.jpg",
      fileSize: externalMsg.fileSize,
    };
  }
  
  // 处理语音消息
  if (externalMsg.type === "voice") {
    media = {
      type: "audio",
      url: externalMsg.voiceUrl,
      mimeType: "audio/mpeg",
      fileName: "voice.mp3",
      duration: externalMsg.duration,      // 语音时长（秒）
    };
  }
  
  // 处理文件消息
  if (externalMsg.type === "file") {
    media = {
      type: "file",
      url: externalMsg.fileUrl,
      mimeType: externalMsg.mimeType,
      fileName: externalMsg.fileName,
      fileSize: externalMsg.fileSize,
    };
  }
  
  await api.receiveMessage({
    channel: "my-channel",
    chatId: externalMsg.chatId,
    senderId: externalMsg.userId,
    body: externalMsg.text || "",         // 媒体消息可能没有文本
    media: media,
    // ... 其他字段
  });
}

6.3 发送媒体消息

outbound: {
  deliveryMode: "direct",
  
  // 发送文本
  sendText: async (params) => {
    const { text, chatId } = params;
    await platformAPI.sendText(chatId, text);
    return { ok: true };
  },
  
  // 发送媒体（可选但推荐实现）
  sendMedia: async (params) => {
    const { type, url, chatId, mimeType, fileName } = params;
    
    switch (type) {
      case "image":
        await platformAPI.sendImage(chatId, url);
        break;
      case "audio":
        await platformAPI.sendVoice(chatId, url);
        break;
      case "video":
        await platformAPI.sendVideo(chatId, url);
        break;
      case "file":
        await platformAPI.sendFile(chatId, url, fileName);
        break;
    }
    
    return { ok: true };
  },
}

8.1 TCP Client Channel（完整版）

import type { PluginAPI } from "openclaw/plugin-sdk/core";
import net from "net";
import { randomUUID } from "crypto";

interface TCPAccountConfig {
  enabled: boolean;
  host: string;
  port: number;
  reconnectInterval?: number;
  authToken?: string;
}

export default function register(api: PluginAPI) {
  const logger = api.logger;
  const config = api.config.get();
  
  const channelId = "tcp-client";
  const sockets = new Map();
  const messageBuffer = new Map(); // 用于处理粘包
  
  // ========== 初始化账户 ==========
  const accounts: Record = 
    config.channels?.[channelId]?.accounts ?? {};
  
  const enabledAccounts = Object.entries(accounts)
    .filter(([_, cfg]) => cfg.enabled)
    .map(([id, cfg]) => ({ id, config: cfg }));
  
  logger.info(`[${channelId}] 初始化 ${enabledAccounts.length} 个账户`);
  
  // ========== 创建 TCP 连接 ==========
  for (const { id: accountId, config: accountCfg } of enabledAccounts) {
    createConnection(accountId, accountCfg);
  }
  
  function createConnection(accountId: string, cfg: TCPAccountConfig) {
    const socket = new net.Socket();
    const reconnectInterval = cfg.reconnectInterval ?? 5000;
    
    let isConnected = false;
    
    const connect = () => {
      logger.info(`[${accountId}] 连接到 ${cfg.host}:${cfg.port}...`);
      socket.connect(cfg.port, cfg.host);
    };
    
    socket.on("connect", () => {
      isConnected = true;
      logger.info(`[${accountId}] ✅ TCP 连接成功`);
      
      // 发送认证信息
      if (cfg.authToken) {
        sendMessage({
          type: "auth",
          token: cfg.authToken,
          client: "openclaw",
        });
      }
    });
    
    socket.on("data", (data) => {
      const chunk = data.toString();
      const buffer = (messageBuffer.get(accountId) || "") + chunk;
      
      // 按换行符分割（处理粘包）
      const lines = buffer.split("\n");
      messageBuffer.set(accountId, lines.pop() || ""); // 保留未完整部分
      
      for (const line of lines) {
        if (!line.trim()) continue;
        
        try {
          const msg = JSON.parse(line);
          handleIncomingMessage(accountId, msg);
        } catch (e) {
          logger.error(`[${accountId}] JSON 解析失败:`, line);
        }
      }
    });
    
    socket.on("close", () => {
      if (isConnected) {
        isConnected = false;
        logger.warn(`[${accountId}] 连接断开，${reconnectInterval}ms 后重连...`);
        setTimeout(connect, reconnectInterval);
      }
    });
    
    socket.on("error", (err) => {
      logger.error(`[${accountId}] 连接错误:`, err.message);
    });
    
    sockets.set(accountId, socket);
    connect();
    
    // 发送消息辅助函数
    function sendMessage(data: any) {
      if (socket.writable) {
        socket.write(JSON.stringify(data) + "\n");
      }
    }
  }
  
  // ========== 处理接收到的消息 ==========
  async function handleIncomingMessage(accountId: string, msg: any) {
    logger.debug(`[${accountId}] 收到消息:`, msg);
    
    switch (msg.type) {
      case "message":
        await api.receiveMessage({
          channel: channelId,
          accountId: accountId,
          chatType: msg.chatType || "direct",
          chatId: msg.chatId,
          senderId: msg.userId,
          senderName: msg.nickname,
          body: msg.text || "",
          messageId: msg.id || randomUUID(),
          timestamp: msg.timestamp || Date.now(),
          media: msg.media,
        });
        break;
        
      case "ping":
        // 心跳响应
        const socket = sockets.get(accountId);
        if (socket?.writable) {
          socket.write(JSON.stringify({ type: "pong" }) + "\n");
        }
        break;
        
      case "error":
        logger.error(`[${accountId}] 服务器错误:`, msg.error);
        break;
    }
  }
  
  // ========== 注册 Channel ==========
  api.registerChannel({
    plugin: {
      id: channelId,
      
      meta: {
        id: channelId,
        label: "TCP Client",
        selectionLabel: "TCP Client (多账户)",
        blurb: "通过 TCP 连接接入外部消息系统",
      },
      
      capabilities: {
        chatTypes: ["direct", "group"],
        media: {
          inbound: ["image", "audio", "file"],
          outbound: ["text"],
        },
      },
      
      config: {
        listAccountIds: (cfg) => 
          Object.keys(cfg.channels?.[channelId]?.accounts ?? {}),
        
        resolveAccount: (cfg, accountId) => 
          cfg.channels?.[channelId]?.accounts?.[accountId ?? "default"],
      },
      
      outbound: {
        deliveryMode: "direct",
        
        sendText: async (params) => {
          const { text, chatId, accountId, senderId } = params;
          const socket = sockets.get(accountId);
          
          if (!socket || !socket.writable) {
            return { ok: false, error: `账户 ${accountId} 未连接` };
          }
          
          const message = {
            type: "reply",
            to: chatId,
            text: text,
            timestamp: Date.now(),
            replyTo: senderId,
          };
          
          socket.write(JSON.stringify(message) + "\n");
          return { ok: true };
        },
      },
    },
  });
  
  // ========== 注册状态命令 ==========
  api.registerCommand({
    name: "tcp-status",
    description: "查看 TCP Client 连接状态",
    handler: () => {
      const statuses = [];
      for (const [accountId, socket] of sockets) {
        statuses.push({
          account: accountId,
          connected: socket.readyState === "open",
          writable: socket.writable,
        });
      }
      
      return {
        text: `TCP Client 状态:\n\n${
          statuses.map(s => 
            `- ${s.account}: ${s.connected ? "🟢 已连接" : "🔴 断开"}`
          ).join("\n")
        }`,
      };
    },
  });
}

8.2 Webhook Channel（HTTP 接收）

import type { PluginAPI } from "openclaw/plugin-sdk/core";
import express from "express";
import bodyParser from "body-parser";

export default function register(api: PluginAPI) {
  const logger = api.logger;
  const config = api.config.get();
  
  const channelId = "webhook-channel";
  const channelConfig = config.channels?.[channelId];
  
  const webhookPort = channelConfig?.webhookPort ?? 3456;
  const webhookPath = channelConfig?.webhookPath ?? "/webhook";
  const secretToken = channelConfig?.secretToken; // 用于验证签名
  
  // 创建 Express 应用
  const app = express();
  app.use(bodyParser.json());
  app.use(bodyParser.urlencoded({ extended: true }));
  
  // 健康检查端点
  app.get("/health", (req, res) => {
    res.json({ status: "ok", channel: channelId });
  });
  
  // Webhook 接收端点
  app.post(webhookPath, async (req, res) => {
    try {
      // 验证签名（可选）
      if (secretToken) {
        const signature = req.headers["x-signature"];
        if (!verifySignature(req.body, signature, secretToken)) {
          res.status(401).json({ error: "Invalid signature" });
          return;
        }
      }
      
      // 解析消息
      const { text, user_id, chat_id, message_id, account_id } = req.body;
      
      if (!text || !user_id) {
        res.status(400).json({ error: "Missing required fields" });
        return;
      }
      
      // 转发给 Gateway
      await api.receiveMessage({
        channel: channelId,
        accountId: account_id || "default",
        chatType: req.body.is_group ? "group" : "direct",
        chatId: chat_id || user_id,
        senderId: user_id,
        senderName: req.body.nickname,
        body: text,
        messageId: message_id || randomUUID(),
        timestamp: Date.now(),
        media: req.body.media,
      });
      
      // 立即返回成功（异步处理）
      res.json({ received: true, messageId: message_id });
      
    } catch (error) {
      logger.error("处理 webhook 失败:", error);
      res.status(500).json({ error: "Internal error" });
    }
  });
  
  // 启动 HTTP 服务器
  const server = app.listen(webhookPort, () => {
    logger.info(`[${channelId}] Webhook 监听在 http://localhost:${webhookPort}${webhookPath}`);
  });
  
  // 注册 Channel
  api.registerChannel({
    plugin: {
      id: channelId,
      
      meta: {
        id: channelId,
        label: "Webhook Channel",
        selectionLabel: "Webhook Channel (HTTP)",
        blurb: "通过 HTTP Webhook 接收消息",
      },
      
      capabilities: {
        chatTypes: ["direct", "group"],
      },
      
      config: {
        listAccountIds: () => ["default"],
        resolveAccount: () => ({ enabled: true }),
      },
      
      outbound: {
        deliveryMode: "direct",
        
        sendText: async (params) => {
          const { text, chatId } = params;
          
          // 这里可以实现回调机制
          // 例如通过另一个 webhook 或轮询队列
          logger.info(`[${channelId}] 需要发送回复到 ${chatId}: ${text}`);
          
          return { ok: true };
        },
      },
    },
  });
  
  // 应用关闭时清理
  process.on("SIGTERM", () => {
    server.close(() => {
      logger.info(`[${channelId}] HTTP 服务器已关闭`);
    });
  });
}

11. 最佳实践

11.1 开发规范

  • 错误处理：始终使用 try-catch 包裹异步操作，避免未捕获的异常导致插件崩溃
  • 重连机制：对于长连接（WebSocket/TCP），实现自动重连逻辑
  • 消息确认：重要的发送操作应等待确认后再返回成功
  • 资源清理：在插件关闭时正确释放资源（关闭连接、停止服务器）

11.2 安全建议

  • 敏感配置（API 密钥、Token）标记为 sensitive: true
  • 验证 Webhook 签名，防止伪造请求
  • 限制可访问的来源 IP（如果平台支持）
  • 使用 HTTPS 而非 HTTP（生产环境）

11.3 性能优化

  • 批量处理消息而非逐条发送（如果平台支持）
  • 缓存账户配置，避免频繁读取
  • 使用连接池管理多个账户连接
  • 对于大文件，考虑使用流式传输
💡 提示
开发 Channel Plugin 时，建议先从一个简单的 Echo Bot 开始，验证基本的消息收发流程，再逐步添加复杂功能。
