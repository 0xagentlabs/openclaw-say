Spoon-Bot

Local-First AI Agent with Native OS Tools, Powered by Spoon-Core
View on GitHub

Project Overview

Spoon-Bot is an advanced local-first AI agent framework that prioritizes native operating system integration. It provides a comprehensive platform for autonomous AI operations with built-in safety mechanisms and extensible tooling.

Core Philosophy

Local Agent Priority: Native OS tools built-in, MCP (Model Context Protocol) as optional extension

4-Layer Memory: File-based + Short-term + Mem0 + Checkpointer

Tool Registry Pattern: Native → Toolkit → MCP (priority order)

Key Features

Multi-Provider LLM

Supports Anthropic, OpenAI, DeepSeek, Gemini, OpenRouter and more

Agent-Centric

Autonomous execution with safety rails

OS-Native

Built-in shell/filesystem tools as priority

Memory-First

Four-layer memory system (file + short-term + Mem0 + checkpointer)

Self-Managing

Self-configuration, self-upgrade, memory management tools

Web3-Enabled

Blockchain operations via spoon-core and spoon-toolkits

Architecture

┌─────────────────────────────────────────────────────────────────┐ │ SPOON-BOT TOOLS │ ├─────────────────────────────────────────────────────────────────┤ │ Native OS Tools (Always Available, Priority) │ │ ├── shell Execute commands (60s timeout, 10KB limit) │ │ ├── read_file Read file contents │ │ ├── write_file Write content to file │ │ ├── edit_file Edit file by replacing text │ │ └── list_dir List directory contents │ ├─────────────────────────────────────────────────────────────────┤ │ Self-Management Tools │ │ ├── self_config get/set/list agent configuration │ │ ├── self_upgrade check updates, install/update skills │ │ └── memory remember, note, search, forget, checkpoint │ ├─────────────────────────────────────────────────────────────────┤ │ Extension Tools │ │ ├── MCP tools Dynamic via configured MCP servers │ │ ├── spoon-toolkit 40+ crypto/blockchain/social tools │ │ └── Skill tools ScriptTool from SKILL.md scripts │ └─────────────────────────────────────────────────────────────────┘

Workspace Structure

~/.spoon-bot/
├── workspace/
│   ├── AGENTS.md        # Agent instructions
│   ├── SOUL.md          # Personality
│   ├── memory/

Gateway API

Spoon-bot includes a WebSocket and REST API gateway for remote agent control.

API Endpoints

Endpoint Method Description
/v1/auth/login POST Authenticate and get tokens
/v1/agent/chat POST Send message to agent
/v1/agent/status GET Get agent status
/v1/ws WS WebSocket for real-time communication

Implementation Phases

  1. Phase 1: Core Agent Loop - Create project structure and implement native OS tools
  2. Phase 2: Session & Memory - Implement 4-layer memory system
  3. Phase 3: CLI & Onboarding - Create Typer CLI and onboarding commands
  4. Phase 4: MCP & Skills - Integrate Model Context Protocol and Skills system
  5. Phase 5: Gateway Mode - Implement WebSocket/REST API gateway
  6. Phase 6: Web3 & Toolkit - Integrate crypto/blockchain tools
  7. Phase 7: Self-Management - Implement self-config and self-upgrade tools
  8. Phase 8: Background Services - Implement cron and heartbeat services

Quick Start

Requirements

Python 3.12+ (required by spoon-ai-sdk)

Installation (with uv)

# Clone the repository
git clone https://github.com/XSpoonAi/spoon-bot.git
cd spoon-bot

# Install with uv
uv sync --all-extras

# Initialize workspace
spoon-bot onboard

# Run in interactive mode
spoon-bot agent

Configuration

Copy .env.example to .env and configure your LLM API keys:

  • ANTHROPIC_API_KEY
  • OPENAI_API_KEY
  • DEEPSEEK_API_KEY
  • GEMINI_API_KEY
  • OPENROUTER_API_KEY
